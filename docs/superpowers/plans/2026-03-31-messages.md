# Messages Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build Slack-like messaging with workspace-wide channels, 1:1 DMs, real-time delivery, typing indicators, online presence, rich text, and file attachments.

**Architecture:** Messages stored in Supabase Postgres with RLS. Real-time via Supabase Realtime (Postgres Changes for messages, Broadcast for typing, Presence for online status). Files in Supabase Storage. Markdown for rich text (sanitized via HTML escaping before rendering). App sidebar shows channels + DMs dynamically.

**Tech Stack:** Next.js 16, Supabase (Realtime, Storage, Postgres), TypeScript, Tailwind CSS, Lucide icons

**Important:** Check `node_modules/next/dist/docs/` for Next.js 16 breaking changes. `params` is a Promise. Use service client (`createServiceClient()`) for operations that bypass RLS. No test runner — verify via browser.

**Security note:** The markdown renderer uses `dangerouslySetInnerHTML` but ALL input is HTML-escaped first via `escapeHtml()` before markdown patterns are applied. Only whitelisted markdown constructs (`<strong>`, `<em>`, `<code>`, `<pre>`, `<a>`, `<br>`) are generated from escaped input. This is safe because the raw user text can never contain unescaped HTML. For production hardening, consider adding DOMPurify as a second layer.
